# stm32f4-ai-register-dev

基于 **STM32F407ZGT6** 的寄存器级裸机开发实验项目，  
使用 **CMSIS** 直接操作硬件寄存器，  
采用 **CMake + 交叉编译工具链** 构建工程，  
并探索 **AI 工具辅助嵌入式开发** 的实际工作方式。

本项目以学习和实验为目的，重点关注：
**AI 在底层嵌入式开发中的可用性、边界与风险**。

---

## 🎯 项目目标

创建本项目的主要目标包括：

- 学习 **STM32F4 寄存器级开发**，不依赖 HAL / LL 库
- 熟悉 **CMSIS** 在裸机工程中的实际使用方式
- 构建一个 **现代化、可复用的嵌入式工程结构**
- 探索 **VSCode + Cline / Roo Code** 等 AI 工具在嵌入式开发中的辅助能力
- 将学习过程、实验代码和经验整理并公开分享

这是一个 **开发实验项目（Dev / Lab）**，而不是量产级框架。

---

## 🧠 为什么选择寄存器级开发？

HAL 库非常成熟且高效，本项目选择寄存器级开发，主要是为了：

- 更深入理解 MCU 硬件工作原理
- 建立寄存器与参考手册之间的直接映射关系
- 避免过度抽象，提升对底层细节的掌控能力
- 更适合验证 AI 在底层代码生成与理解方面的实际表现

这不是对 HAL 的否定，而是另一种学习路径。

---

## 🤖 关于 AI 的使用说明

本项目有意识地引入 AI 工具，用于：

- 生成寄存器初始化代码草稿
- 辅助理解参考手册中的复杂描述
- 重构和整理底层代码结构
- 减少重复性样板代码的编写时间

> **AI 仅作为辅助工具使用，  
> 所有寄存器配置和关键逻辑均需人工验证。**

---

## 🛠 开发环境与工具链

- MCU：**STM32F407ZGT6**
- 内核：**ARM Cortex-M4F**
- 编译器：**arm-none-eabi-gcc**
- 构建系统：**CMake**
- 编辑器：**VSCode**
- AI 工具：
  - Cline
  - Roo Code
- 下载与调试：
  - OpenOCD
  - ST-Link
